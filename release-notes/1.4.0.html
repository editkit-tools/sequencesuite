Release notes

SequenceSuite 1.4.0

Clip names you can trust, a consolidated Source Pull report for VFX and grading handoffs, and a camera-report style ordering preset - plus a more resilient way of getting damaged files open in the first place.

Honest clip names

  • Source File Name and Clip Name show what's actually on disk, not the bin name from the NLE.
  • Media that never had a real file - renders, VO recordings, Avid subclips - is labelled by its clip name instead.
  • Where an EDL only carries the real filename in its reel column (a common export quirk), SequenceSuite spots the match against the clip name and restores the proper filename, underscores and all.

Source Pull report

  • A new Source Pull report consolidates every use of a source clip across a sequence into one row per source range - the shape a VFX, grading or archive pull actually needs.
  • Overlapping and adjacent uses of the same source are merged, with a configurable gap tolerance and configurable handles either side.
  • Available as an Excel and CSV report from the Sections list, with its own options in the Custom Export window.

C MODE ordering

  • A new "C MODE" row order in the Custom Export window sorts by reel, then source timecode, then sequence timecode - the camera-report ordering some post houses standardise on for delivery.
  • Ordering only - nothing about which clips appear or how they're grouped changes.

Getting damaged files open

  • A file with the wrong extension is caught and rerouted - drop in an EDL that got renamed to .xml and SequenceSuite still finds the right way to read it.
  • Genuinely damaged files can now be repaired - truncated XML, bad encoding, stray invalid characters - with a clear "Attempt Repair" offer. Repairs never touch your original file, and a repaired load is always flagged with a note.
  • A standalone File ▸ Repair File… window lets you triage a damaged EDL, XML or OTIO file on its own.

Fixes and polish

  • The XLSX front page (Welcome sheet / Sequences index) can be switched off from the Custom Export window's Options, for workbooks that go straight to production.
  • Batch exports keep every file when two sequences in one project share a name - the second gets a "(2)" suffix.
  • Search works in Gather mode too, filtering the media table just like Sheet.
  • Dupe filters and expanded groups survive switching between modes.
  • Workbooks and CSVs open cleanly in Excel, accented characters and all.
  • Gathered copies keep their creation dates, permissions and Finder tags.
  • Locate matches media regardless of filename capitalisation, and cancelling thumbnail generation stops it immediately.
  • A deep round of under-the-bonnet stability work makes long parses, gathers and exports more resilient while you keep working.
